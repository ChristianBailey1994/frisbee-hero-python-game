Work Section:

We spent approximately 8 hours working on FrisbeeHero thus far.

One of our biggest victories was getting the main gameplay loop working. The player can
move, throw frisbees, destroy enemies, earn points, and either win or lose based on what
happens during the round. Another victory was building useful systems like image loading,
centered drawing, object creation, enemy movement, collision detection, score tracking,
and on-screen messages. Some of these items such as the collision detection were assisted
by the use of AI.

Some of our main challenges were getting image file paths to work correctly, figuring
out collision logic, and trying to improve the overall gameplay. Our code also shows ideas
we wanted to add later, like better player action images, randomized enemy formations,
different enemy types, more backgrounds, a curved frisbee path, and improved title or win
graphics.

Working with a partner had both pros and cons. A major benefit was being able to share ideas,
and help each other solve problems. A challenge was making sure work was divided
evenly and that both people stayed on the same page. In the future, we could improve by planning
responsibilities and work times that both can attend which would result in more progress of our game.
